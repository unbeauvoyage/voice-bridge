## Learnings
- Agent teams (TeamCreate/SendMessage) lose context when CLI dies — team leads are persistent `-p` sessions instead
- `claude -p --session-id` + `--resume` gives persistent context with no token waste on reload
- Codex must always run in background — never block the calling agent
- `cmux new-workspace --command "claude --resume $ID --remote-control"` opens session interactively
- `cmux close-workspace` closes view but session survives on disk
- Session naming: `meta:*` prefix for meta-managed, prevents mixing with user sessions
- Research agents must append findings immediately — no data loss acceptable for customer work
- `.worklog/` folder in every project for agent work logs, globally gitignored
- `claude -p` doesn't support `-C` flag — must `cd` to project dir first
- `claude -p` team lead sessions can't effectively spawn subagents that write files — run agents directly from meta-manager instead
- `claude -p --resume` background messages produce zero output and don't reliably trigger actions — use `cmux send` to workspace instead
- Agent teams (TeamCreate) die on session end — must be re-created on every resume. Team leads need startup protocols.
- `/usage` is a built-in CLI command, not a skill — COMMAND cannot call it programmatically. Need OTel for automated token tracking.
- Claude Code Channels (v2.1.80+) replace terminal injection for inter-agent messaging. Plugins with `claude/channel` capability push MCP notifications into sessions. Requires `--dangerously-load-development-channels` for custom plugins not on Anthropic's allowlist. Channel messages wake idle sessions — no Enter needed.
- `--plugin-dir` loads plugins as MCP servers but NOT as channels. Must use `--channels` or `--dangerously-load-development-channels` for channel capability to activate.
- Plugin naming: `claude-channel-*` prefix in package.json name follows Anthropic's convention (e.g., `claude-channel-fakechat`).
